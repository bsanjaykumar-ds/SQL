# SQL Practice — Employee Database

A short SQL practice script that walks through core database and table
operations using a simple `employees` table: creating a database, building
a table, modifying its structure, and running basic CRUD queries.


The script runs through, in order:

1. **Database & table creation** — creates `MYDB` and an `employees` table
   (`employees_id`, `first_name`, `last_name`, `hourly_pay`, `hire_date`).
2. **Table/column changes** — renaming a table, adding a column
   (`phone_number`), renaming it to `email`, changing its data type, and
   repositioning it with `MODIFY ... AFTER`.
3. **Inserts** — a multi-row `INSERT`, plus a partial-row insert that only
   sets `employees_id` and `first_name`.
4. **Queries** — `SELECT` with `WHERE` filters covering equality, `>=`,
   `<=`, `!=`, and `IS NULL`.
5. **Updates & deletes** — setting a column to `NULL`, disabling
   `SQL_SAFE_UPDATES`, and deleting a row.
6. **Transactions** — a `ROLLBACK`, toggling `autocommit`, and a `COMMIT`.

## Requirements

- MySQL 8.0+ (the script uses `ALTER TABLE ... RENAME COLUMN`, which
  requires MySQL 8.0 or later).
